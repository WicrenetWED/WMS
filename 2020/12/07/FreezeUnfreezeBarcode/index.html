8&9冻结解冻不良品

8.1 接口说明

项目 描述
功能 冻结条码、解冻不良品条码
数据方向 WMS–>LMS
实时性 实时
说明 冻结时条码状态需为“正常”， 解冻时条码状态需为已冻结或者已完成异常品申报流程

8.2 请求URL与方式

Url:/api/FreezeUnfreezeBarcode

测试地址：http://139.219.15.177/BPServiceHub/api/FreezeUnfreezeBarcode

方法：Post

协议：http

请求头参数

参数 描述 类型 是否必填 备注
Token 令牌，用于认证 String 是

请求体Body参数

参数 描述 类型 是否必填 备注

8.7 相关业务知识

冻结，解冻，LMS的条码有两个状态，一个是库存状态，还有一个是质量状态。这里讲的冻结、解冻是质量的条码状态。传过来冻结的前提，必须是LMS里这个条码是“正常”状态，传过来解冻的前提是LMS里这个条码是“已冻结”状态，除此之外的其他状态，都会被我们接口报错返回。“不良品”状态是LMS操作完成了不良品申报后，变的状态。“不良品”状态是不允许冻结、解冻的。必须通过申报后恢复成“正常”。所以，如果你们的WMS在设计的时候也要考虑到“不良品”这个状态的设置和达到的控制点。因为“不良品”是在LMS里操作的，所以可能你们的WMS里可能会少一个“不良品”的状态。但好在“不良品”状态是给后续QC处理意见（LMS上操作使用的），如果你们WMS缺少这个“不良品”状态，也可以用条码所在的虚拟仓（不良品仓）来控制发货。
